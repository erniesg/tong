# Interaction Modes Contract

## Hangout mode
Purpose:
- First-person immersive validation of mastery objectives in narrative context.

Screen policy:
1. Show only character dialogue and Tong hints/tips.
2. Keep objective tracking in-state and non-intrusive.
3. Do not surface debug or meta controls during active turns.

Session policy:
1. Must be tied to one objective ID.
2. Agentic tool usage is allowed internally, but output remains in-character.
3. Scene reacts to user responses and updates objective progress every turn.

## Learn mode
Purpose:
- Structured instruction and drilling for targeted progression.

Screen policy:
1. Chatbox UI styled per city social app aesthetic.
2. Must expose "View previous sessions".
3. Must expose "Start new session".

Session policy:
1. Every learn session must have objective ID + level + targets.
2. New sessions can preload personalized targets from recent media.
3. Completed sessions feed readiness for next hangout and mission gates.
